Hexo的布局

Hexo 有三种默认布局：post（博文）、page（页面） 和 draft（草稿），
它们分别对应不同的路径，page是直接储存在source下的。post和draft对应不同的子文件夹。
自定义的其他布局和post储存在同一文件夹。

Post

类似本页面。

Page

类似About（关于我）页面。
参照：关于我

Draft

不会公开的部分。
使用

可以将它公开（移入Post这一类别）。
在执行时加上 –draft 参数，或是把render_drafts参数设为true 可以预览草稿。

新文章格式的设定

Front-matter 是文件最上方以 — 分隔的区域，用于指定个别文件的变量。

Hexo官方文档Front-matter
参数 描述 默认值
layout 布局
title 标题
date 建立日期 文件建立日期
updated 更新日期 文件更新日期
comments 开启文章的评论功能 true
tags 标签（不适用于分页）
categories 分类（不适用于分页）
permalink 覆盖文章网址
分类和标签
类别 区别
分类 不能同时在同级的分类下存在。
标签 可以同时存在多个标签。

Hexo生成文件名的设定

可以靠改变new_post_name来变更默认的文件名。

变量 描述
:title 标题（小写，空格将会被替换为短杠）
:year 建立的年份，比如， 2015
:month 建立的月份（有前导零），比如， 04
:i_month 建立的月份（无前导零），比如， 4
:day 建立的日期（有前导零），比如， 07
:i_day 建立的日期（无前导零），比如， 7

模板的使用

在新建文章时，Hexo 会根据scaffolds文件夹内相对应的文件来建立文件。

此处的photo就是作为模板的文件。

变量 描述
layout 布局
title 标题
date 文件建立日期
